How to read this guide
- All commands are slash commands typed in chat.
- Command names are case-insensitive.
- Use <required> for required parameters and [optional] for optional ones.
- If a command takes several positional arguments, quote names with spaces when needed.
- Common target shortcuts:
  me = your active character
  here = your current structure or solar system, depending on the command


HELP, CHAT, AND BROADCAST
-------------------------

/help, /commands, /commandlist
What it does: Shows the built-in command list.
Parameters: none.
Example: /help

/motd
What it does: Shows the current MOTD block.
Parameters: none.
Example: /motd

/deer_hunter
What it does: Shows the Deer_Hunter thank-you message and tries to play a brief microjump-style effect.
Parameters: none.
Example: /deer_hunter

/blue
/red
/teal
/yellow
What they do: Change and persist your chat color/presence role.
Parameters: none.
Examples:
- /blue
- /yellow

/announce <message>
What it does: Sends a system message to every connected session.
Parameters:
- <message> = the text to broadcast
Example: /announce Server restart in 5 minutes.


STATUS, INSPECTION, AND SESSION INFO
------------------------------------

/where
What it does: Shows whether you are docked or in space, plus the current system/location ID.
Parameters: none.
Example: /where

/who
What it does: Lists all connected characters.
Parameters: none.
Example: /who

/session
What it does: Dumps a compact session summary: character, ship, corp, docked location, system, wallet.
Parameters: none.
Example: /session

/hangar
What it does: Shows your active ship and the ships in your current ship hangar.
Parameters: none.
Notes: Most useful while docked.
Example: /hangar

/wallet, /isk
What they do: Show ISK balance, PLEX balance, EverMarks, and last ISK delta.
Parameters: none.
Examples:
- /wallet
- /isk

/evermarks
What it does: Shows your character EverMarks balance.
Parameters: none.
Example: /evermarks

/corpevermarks
What it does: Shows your corporation EverMarks balance.
Parameters: none.
Example: /corpevermarks

/iteminfo <itemID>
What it does: Shows the raw item row for a specific itemID.
Parameters:
- <itemID> = exact item ID
Example: /iteminfo 1029384756
Note: This is the real inspection command. The internal handler still prints a legacy usage string that says /item <itemID>, but /item is actually an alias of /giveitem.

/typeinfo <ship name|typeID>
What it does: Shows ship type metadata: typeID, groupID, categoryID, and published flag.
Parameters:
- <ship name|typeID> = exact ship name or ship typeID
Example: /typeinfo Raven


TRAVEL, MOVEMENT, FX, AND SYSTEM LOADING
----------------------------------------

/dock
What it does: Jumps you to your home/clone station.
Parameters: none.
Example: /dock

/solar <system name>
What it does: Teleports your session to another solar system and updates session identity/local membership.
Parameters:
- <system name> = solar system name
Example: /solar Jita

/tr <me|characterID|entityID> <destination|pos=x,y,z|offset=x,y,z>
What it does: Transports a character session or runtime entity.
Parameters:
- <me|characterID|entityID> = who or what to move
- <destination> = a destination token the transport parser can resolve
- pos=x,y,z = absolute coordinates
- offset=x,y,z = offset from the current/target context
Examples:
- /tr me Jita
- /tr me pos=0,0,0
- /tr me offset=0,0,5000
- /tr 90000001 offset=0,0,10000
Notes:
- Session targets can cross stations/systems.
- Runtime entities are point-move only and must stay in their current solar system.

/effect <name>
/effect stop
/effect stop <name>
What it does: Plays or stops self FX on your ship.
Parameters:
- <name> = effect key from the playable effect registry
Examples:
- /effect microjump
- /effect stop
- /effect stop microjump
Notes:
- /effect with no argument, /effect list, /effect help, and /effect ? show the available effect list.
- You must be in space.

/loadsys
What it does: Preloads the destination systems for all stargates in your current system.
Parameters: none.
Example: /loadsys

/loadallsys
What it does: Preloads every solar system scene.
Parameters: none.
Example: /loadallsys

/tidi [0.1-1.0|auto]
What it does: Shows current TiDi when no argument is given, sets a manual TiDi override with a factor, or returns the system to autoscaling.
Parameters:
- [0.1-1.0] = manual TiDi factor
- [auto] = clear manual override and return to autoscaling
Examples:
- /tidi
- /tidi 0.5
- /tidi auto

------------------------------------------------------

/heal

What it does: Ensures all unpublished/GM skills are trained to level V for the current character.
Parameters: none.
Example: /gmskills

/giveskill <target> <skill name|typeID|all|super> [level]
/giveskills <target> <skill name|typeID|all|super> [level]
What they do: Grant one skill, all published skills, or all published+unpublished skills to a target.
Parameters:
- <target> = me, characterID, or exact character name

- super = all published and unpublished skills

Examples:
- /giveskill me "Gallente Cruiser" 5
- /giveskill me all
- /giveskills 90000001 super 4

/removeskills <target> <skill name|typeID|all>
What they do: Remove one skill or remove all published skills from a target.
Parameters:
- <target> = me, characterID, or exact character name
- <skill name|typeID|all> = exact skill or all published skills
Examples:
- /removeskill me "Gallente Cruiser"
- /removeskills 90000001 all

SPACE OBJECTS, DEBRIS, DUMMIES, AND DEATH TESTS
-----------------------------------------------

/jetcan <item name|typeID> [amount]
What it does: Creates a jetcan in space and moves the chosen item stack into it.
Parameters:
- <item name|typeID> = item to jettison
- [amount] = quantity, default 1
Example: /jetcan Tritanium 5000
Notes:
- You must be in space.
- The jetcan expires in 2 hours.

/container <name|typeID> [count]
/spawncontainer <name|typeID> [count]
What they do: Spawn space containers within about 20 km.
Parameters:
- <name|typeID> = debris/container type
- [count] = number to spawn
Examples:
- /container list
- /container "Cargo Container" 5
- /spawncontainer 23 10
Notes:
- /container list shows all valid types.
- You must be in space.

/wreck <name|typeID> [count]
/spawnwreck <name|typeID> [count]
What they do: Spawn wrecks within about 20 km.
Parameters:
- <name|typeID> = wreck type
- [count] = number to spawn
Examples:
- /wreck list
- /wreck "Wreck" 10
- /spawnwreck 186 3
Notes:
- /wreck list shows all valid wreck types.
- You must be in space.

/testclear
What it does: Clears nearby wrecks/containers within about 20 km.
Parameters: none.
Example: /testclear
Notes: You must be in space.

/sysjunkclear
What it does: Clears wrecks/containers across the entire current solar system.
Parameters: none.
Example: /sysjunkclear
Notes: You must be in space.

/keepstar
What it does: Spawns a runtime-only Palatine Keepstar visual roughly 400 km away.
Parameters: none.
Example: /keepstar
Notes:
- Visual only.
- Not persisted.
- Clears on restart.

/fire [ship name|typeID]
What it does: Spawns one combat dummy hull roughly 20 km away.
Parameters:
- [ship name|typeID] = optional hull to spawn, defaults to Drake
Examples:
- /fire
- /fire Raven

/fire2 [count]
What it does: Spawns a staggered arrowhead fleet of combat dummy ships ahead of you.
Parameters:
- [count] = optional fleet size
Examples:
- /fire2
- /fire2 25

/deathtest [ship name|typeID] [count]
What it does: Spawns a temporary field of hulls across about 20 km, then detonates them into wrecks after a short delay.
Parameters:
- [ship name|typeID] = optional hull to use; defaults to your current ship type if available
- [count] = optional amount
Examples:
- /deathtest
- /deathtest Raven 12
Notes: You must be in space.

/suicide
What it does: Self-destructs your active ship, creates a wreck, and places you in a capsule.
Parameters: none.
Example: /suicide
Notes: You must be in space.

/deadwarp
What it does: Forces an immediate warp start to a safe point several AU away.
Parameters: none.
Example: /deadwarp
Notes: You must be in space.


NPCS, MINING FLEETS, CRIMEWATCH, AND SECURITY
---------------------------------------------

/npc [amount] [faction|profile|pool]
What it does: Spawns transient NPCs near you, preferring authored pools.
Parameters:
- [amount] = number to spawn, default 5
- [faction|profile|pool] = optional authored selection query
Examples:
- /npc
- /npc 10 guristas
- /npc 6 generic_hostile

/mnpc [amount] [faction|profile|pool]
What it does: Spawns missile-capable transient NPCs near you.
Parameters:
- [amount] = number to spawn, default 5
- [faction|profile|pool] = optional authored selection query
Examples:
- /mnpc
- /mnpc 8 npc_missile_hostiles

/npctest [amount]
What it does: Spawns stationary authored combat NPCs in a sphere about 20 km ahead of your ship.
Parameters:
- [amount] = number to spawn
Examples:
- /npctest
- /npctest 12
Notes: Short-range hulls are range-clamped inward so they can still shoot while stationary.

/npctest2 [amount]
What it does: Spawns stationary NPCs that fight each other until one side wins or they mutually destroy each other.
Parameters:
- [amount] = number to spawn, minimum 2
Examples:
- /npctest2 10
- /npctest2 20

/npcw [amount] [profile|pool]
What it does: Spawns transient NPCs that warp in to you.
Parameters:
- [amount] = number to spawn
- [profile|pool] = optional authored selection query
Examples:
- /npcw
- /npcw 6 guristas

/wnpc [amount] [profile|pool]
What it does: Same as /npcw, but also applies a chase propulsion override to the warped NPCs when possible.
Parameters:
- [amount] = number to spawn
- [profile|pool] = optional authored selection query
Examples:
- /wnpc
- /wnpc 6 guristas

/concord [amount] [profile|pool]
What it does: Spawns transient CONCORD near you.
Parameters:
- [amount] = number to spawn
- [profile|pool] = optional authored selection query
Examples:
- /concord

/npcclear <system [npc|concord|all]|radius <meters> [npc|concord|all]>

Parameters:
- system [npc|concord|all] = clear the current system
- radius <meters> [npc|concord|all] = clear only around you
Examples:
- /npcclear system all
- /npcclear radius 50000 npc

/npcminer [amount] [profile|pool|group]

- [amount] = fleet size

UPWELL COMMAND FAMILY
---------------------

/upwell help
What it does: Shows the Upwell subcommand list.
Example: /upwell help

/upwell list [all]
What it does: Lists known structures.
Parameters:
- [all] = include all systems; without it, the list is scoped to the current system
Examples:
- /upwell list
- /upwell list all

/upwell info <structureID|name|here>
What it does: Shows a formatted summary for one structure.
Examples:
- /upwell info here
- /upwell info 1030000000001

/upwell seed <astrahus|fortizar|keepstar|raitaru|azbel|sotiyo|athanor|tatara|palatine> [name]
What it does: Seeds a structure in the current system near your ship.
Examples:
- /upwell seed astrahus
- /upwell seed fortizar "Fort Elysian"

/upwell anchor <structureID|name|here>
What it does: Forces a structure into its anchored/runtime-synced state.
Examples:
- /upwell anchor here
- /upwell anchor 1030000000001

/upwell core <structureID|name|here> <on|off>
What it does: Installs or removes the quantum core state.
Examples:
- /upwell core here on
- /upwell core 1030000000001 off

/upwell ff <structureID|name|here> <seconds>
What it does: Fast-forwards structure timers by a number of seconds.
Examples:
- /upwell ff here 3600
- /upwell ff 1030000000001 86400

/upwell repair <structureID|name|here>
What it does: Fully repairs the structure.
Examples:
- /upwell repair here
- /upwell repair 1030000000001

/upwell state <structureID|name|here> <state>
What it does: Sets the raw structure state.
Parameters:
- <state> = the state token accepted by the structure state handler
Examples:
- /upwell state here online
- /upwell state 1030000000001 unanchored

/upwell upkeep <structureID|name|here> <full_power|low_power|abandoned>
What it does: Sets the upkeep state.
Examples:
- /upwell upkeep here low_power
- /upwell upkeep 1030000000001 abandoned

/upwell service <structureID|name|here> <serviceID|serviceName> <online|offline>
What it does: Toggles an individual structure service.
Examples:
- /upwell service here clone online
- /upwell service 1030000000001 5 offline

/upwell damage <structureID|name|here> <shield|armor|hull|kill> [amount]
What it does: Applies GM damage to a structure.
Parameters:
- <shield|armor|hull|kill> = layer or kill shortcut
- [amount] = optional numeric damage amount
Examples:
- /upwell damage here shield 500000
- /upwell damage 1030000000001 kill

/upwell kill <structureID|name|here>
What it does: Immediately destroys the structure.
Examples:
- /upwell kill here
- /upwell kill 1030000000001

/upwell timer <structureID|name|here> <scale>
What it does: Sets structure timer scale.
Examples:
- /upwell timer here 0.1
- /upwell timer 1030000000001 10

/upwell tether <status|clear|scram|cyno|fighters|fw|delay> [value]
What it does: Inspects or edits tether restriction debug state for the current character.
Parameters:
- status = show current restriction state
- clear = clear all restrictions
- scram|cyno|fighters|fw <on|off> = toggle that restriction
- delay <seconds> = apply tether delay
Examples:
- /upwell tether status
- /upwell tether scram on
- /upwell tether fighters off
- /upwell tether delay 30

/upwell remove <structureID|name|here>
What it does: Removes the structure from persisted state.
Examples:
- /upwell remove here
- /upwell remove 1030000000001

/upwell purge [all]
What it does: Bulk-removes persisted structures.
Parameters:
- [all] = purge every system; without it, purge only the current system
Examples:
- /upwell purge
- /upwell purge all

/upwell safety <structureID|name|here> [char|corp]
What it does: Moves personal or corporation assets from the structure into an asset safety wrap.
Parameters:
- [char|corp] = owner kind, default char
Examples:
- /upwell safety here
- /upwell safety 1030000000001 corp

/upwell wraps [char|corp]
What it does: Lists asset safety wraps for the current character or corporation.
Parameters:
- [char|corp] = owner kind, default char
Examples:
- /upwell wraps
- /upwell wraps corp

QUICK REMINDERS
---------------

- Docked-only commands usually involve hangars, fitting, or inventory grants.
- In-space-only commands usually involve NPCs, FX, transport, debris, combat dummies, or runtime visuals.
- /item is a grant command, not an inspection command.
- The structure, sovereignty, and automation families are large enough that it is worth using their help forms first:
  /upwell help
  /upwellauto help
  /sov help
  /sovauto help
