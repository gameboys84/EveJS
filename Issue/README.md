## 二、问题管理规范

### 2.1 文件命名

```
NNN-<简短英文描述>.md
```

- `NNN` 为三位数字编号，从 001 开始递增
- 简短描述使用英文小写 + 连字符

### 2.2 文档定位

每个问题文档包含该问题的**完整信息**：背景、复现条件、根因分析、修复方案。读者无需跨文件拼凑。

### 2.3 问题文档结构

```markdown
# 问题 #NNN: <简短标题>

### 基本信息
| 字段 | 内容 |
|------|------|
| **问题编号** | #NNN |
| **发现日期** | YYYY-MM-DD |
| **影响版本** | EveJS v0.X |
| **严重程度** | 高 / 中 / 低 |
| **状态** | 🔍 待验证 / 🔄 修复中 / ✅ 已解决 |
| **类别** | 掉落系统 / 战斗系统 / 网络 / 数据 / 其他 |

### 问题描述
<现象、预期行为、实际行为、复现步骤>

### 根因分析
<按可能性排序的假设 + 依据>

### 排查过程
<已执行的排查步骤和结论>

### 修复方案
<方案描述 + 代码示例>

### 相关文件
<关联的 Doc/ 或代码文件路径>
```

### 2.4 状态说明

| 图标 | 含义 |
|------|------|
| 🔍 待验证 | 问题已记录，尚未确认根因 |
| 🔄 修复中 | 已定位原因，正在修复 |
| ✅ 已解决 | 问题已修复并验证通过 |
| ⏸️ 暂缓 | 暂时搁置，后续处理 |
| ❌ 无法复现 | 无法重现问题，关闭 |

### 2.5 严重程度

| 级别 | 说明 |
|------|------|
| 高 | 影响核心玩法或导致功能不可用 |
| 中 | 影响体验但有替代方案 |
| 低 | 轻微问题，不影响主要功能 |

## 四、维护规则

1. **新增问题**: 复制模板 → 创建新文件 → 更新本文件的索引表
2. **更新进度**: 直接在对应问题文件中修改内容和状态
3. **问题解决**: 将状态改为 ✅ 已解决，补充最终修复方案
4. **跨问题关联**: 在相关问题的"相关文件"中互相引用

### 5.4 调试方法论

由于没有客户端代码，调试主要依赖：

1. **服务端日志**: 使用项目标准日志格式 `log.info("[Tag] key=value")`
2. **行为对比**: 对比不同版本间的日志输出差异
3. **数据分析**: 分析静态数据表理解 CCP 的数据结构
4. **社区参考**: 参考 Discord 社区 (https://discord.gg/KMuJrMDEBa) 的经验
5. **视频对比**: Movie/ 目录下有 CCP 与 EveJS 的界面对比录像
