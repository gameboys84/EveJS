# EveJS 问题跟踪 (Issue Tracking)

> 本目录用于本地跟踪 EveJS 项目开发中发现的各类问题。每个问题独立一个文件，方便单独查阅和更新。

---

## 目录结构

```
Issue/
├── README.md                         # 本文件：说明 + 问题索引 + 模板
├── 001-wreck-loot-not-pickable.md   # 问题 #001: 残骸无法拾取物品
├── 002-<简短描述>.md                 # 问题 #002 (新增问题时命名)
└── ...
```

## 文件命名规范

```
NNN-<简短英文描述>.md
```

- `NNN` 为三位数字编号，从 001 开始递增
- 简短描述使用英文小写 + 连字符，概括问题核心
- 例如: `001-wreck-loot-not-pickable.md`

### 状态说明

| 图标 | 含义 |
|------|------|
| 🔍 待验证 | 问题已记录，尚未确认根因 |
| 🔄 修复中 | 已定位原因，正在修复 |
| ✅ 已解决 | 问题已修复并验证通过 |
| ⏸️ 暂缓 | 暂时搁置，后续处理 |
| ❌ 无法复现 | 无法重现问题，关闭 |

| 级别 | 说明 |
|------|------|
| 高 | 影响核心玩法或导致功能不可用 |
| 中 | 影响体验但有替代方案 |

---

- 预期行为是什么

**依据**: ...

### 建议排查步骤

1. <具体操作或代码>

## 维护规则

1. **新增问题**: 复制模板 → 创建新文件 → 更新本文件的索引表
2. **更新进度**: 直接在对应问题文件中修改内容和状态
3. **问题解决**: 将状态改为 ✅ 已解决，补充最终修复方案
4. **跨问题关联**: 在相关问题的"相关文件"中互相引用
